2. Mimari ve Dosya İlişkileri

Üretim tarafı

couplateV2.py, GIS katmanlarını okuyup spatial join yapar ve look-ups klasöründeki Excel sözlüklerini üretir. Bu aşama ağır ön işleme tarafıdır.

Trace tarafı

graph.py, import_dicts.py üzerinden hazır lookup dosyalarını okur; graph kurar; başlangıç hücresinden downstream trace çıkarır.

4. Ana Çağrı Sırası

  1. Graph() nesnesi oluşturulur ve sınıf alanları boş/varsayılan değerlerle hazırlanır.
  2. init_dicts(), look-ups Excel dosyalarını import_dicts.load_and_unpack_all() ile okur.
  3. make_graph(), point_df ve edge_df üzerinden NetworkX Graph oluşturur.
  4. search_and_create_tree(start_hucre_id), başlangıç hücresini trace eder.
  5. preperation(), başlangıç hücresinde trace edilecek ayırıcı koordinatlarını hazırlar.
  6. pre_process(), kaynak ayırıcıdan belirli cutoff ile tüm kısa yolları üretir.
  7. process_paths_fast(), downstream ve kablo koşullarını sağlayan en uygun adayı seçer.
  8. prepare_connection(), trace sonucunu tabloya ekler ve alt merkezin içinden devam edilecek yeni queue elemanlarını üretir.
  9. Döngü bitince sonuçlar Excel'e yazılır.

6. Trace Karar Mantığı

process_paths_fast filtreleri

  1. Path, bağlantı başlangıcı veya bitişinden geçmeli.
  2. Path uzunluğu en az 4 node olmalı.
  3. Target merkez, source merkezden farklı olmalı.
  4. Path edge id'leri güvenli şekilde bulunmalı.
  5. Path içinde en az bir HAT/kablo edge id olmalı.
  6. Target ayırıcı koordinatıysa alt_merkez, değilse sonlanan_hat adayı olur.
  7. Alt merkez adayı varsa öncelik ondadır; en kısa ham path seçilir.
  8. Alt merkez yoksa sonlanan hat adaylarından en uzun ham path seçilir.

prepare_connection kararları

  1. Satırdan context çıkarılır.
  2. Sonuç satırı hem ana tabloya hem debug tablosuna eklenir.
  3. sonlanan_hat ise queue genişletilmez.
  4. alt_merkez root merkez ise queue genişletilmez.
  5. Alt merkez tek bara ise trace_merkez_ici_1_bara çağrılır.
  6. Alt merkez 2.5 bara ise trace_merkez_ici_2_5_bara çağrılır.
Önemli: Yeni ana akışta path_ids liste olarak tutulur. Eski legacy fonksiyonlarda string liste davranışı korunmuştur. get_mesafe iki tipi de kabul eder.
